# Step 6.1.5：デザインシステム（DS）の有無（Step 6で「1」を選択した場合のみ実行）

対象 Figma に **体系化されたデザインシステム**（Variables / Styles が Config/Theme 層構造で整備されているもの）がありますか？

[1] あり（DS 駆動プロジェクト）
[2] なし（Figma はレイアウト参照のみ）
[3] 不明 / これから作る

### [1] を選択した場合、以下を入力：

1. DS ファイル名（例：NDS_v2 Design System）
2. DS ファイルの File Key または URL
3. DS ファイルと対象デザインが同一ファイルか / 別ファイルか

※ 回答内容は `design-system/figma-reference.md` に記録します。
※ 以降、実装は **Theme トークン経由のみ**（生hex / 生px 禁止、`32_generation_rules.md` 8-6 参照）。
※ 命名衝突時は `68_token_bridge.md` の対応表を参照し、Figma Theme 命名を正とします。

### [3] を選択した場合：

- 新規作成の場合は `67_design_system_workflow.md` の4ステップに従う
- 本プロジェクトで DS を作るかどうかは Mode 決定時に再確認
