# 03_os_inbox.md — AI-OS 更新インボックス

## 参照タイミング
- AI がセッション中に OS 候補を検知したとき（書き込み）
- 正式 OS アップデートセッション実施時（読み込み・統合）

---

## このファイルの役割

```
実作業中の気づき
    ↓ AI が検知 → 「インボックスに追加しますか？」
03_os_inbox.md（ここ）← ステージング置き場
    ↓ 正式 OS アップデートセッション（人間が明示依頼）
各 _ai-os/XX.md へ統合
```

インボックスは **ステージングエリア**。直接 OS を書き換えるのではなく、
一度ここに溜めてから人間がレビューして正式統合する。

---

## 書き方（AI が自動追記する形式）

**新規ルール・思想（OS候補）**
```
## YYYY-MM-DD #連番 [OS候補]
- 対象：XX_filename.md（新規の場合は「新規: XX_name.md」）
- 内容：追加・変更したいルールや思想
- きっかけ：何があってこれに気づいたか
- ステータス：未処理
```

**既存ルール間の矛盾（OS矛盾）**
```
## YYYY-MM-DD #連番 [OS矛盾]
- 矛盾箇所：XX.md「〜」 vs YY.md「〜」
- 今回の対応：どちらを優先したか / 保留
- 推奨解決策：（AI の見解）
- ステータス：未処理
```

---

## 正式 OS アップデートセッションの実施タイミング

以下のいずれかで「正式アップデートしよう」と判断する：

| 条件 | 目安 |
|---|---|
| 未処理メモが溜まった | 3件以上 |
| 大きなプロジェクトが一段落した | 節目ごと |
| 同じ判断を複数回繰り返した | 2回目以降 |

### 正式アップデートの依頼文

> `_ai-os/03_os_inbox.md` の未処理メモをレビューして、AI-OS の各ファイルに統合してください。
> 既存ルールと矛盾する場合は統合前に確認してください。
> 統合後はステータスを「統合済み」に更新してください。
