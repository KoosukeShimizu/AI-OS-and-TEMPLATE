# 54_decision_log_format.md

## 参照タイミング
重要判断・例外・破壊的変更の可能性が出た場合のみ参照する（decision_log記録時）。

## このファイルについて
重要な設計判断・ルール衝突・例外採用を記録するためのログである。  
30_conflict_resolution.md に従い、必要に応じて記録すること。

目的：
- 同じ議論を繰り返さない
- 判断の背景を明確にする
- 例外の増殖を防ぐ

Updated: 2026-02-20
---

# 記録フォーマット

## YYYY-MM-DD

### 1. 衝突点 / 判断対象
（例：再利用性 vs 実装スピード）

### 2. 参照したルール階層
（例：Level1 ai_context.md / Level2 20_mindset.md）

### 3. 選択した方針
（どちらを優先したか）

### 4. 理由
（なぜその判断をしたか）

### 5. 影響範囲
（どのページ / どのコンポーネント）

### 6. 一時的か恒久的か
（今回限り / 今後も適用）

### 7. ai_context.md への反映有無
（反映した / していない / 不要）

---

# ルール

- 軽微な判断は記録しない
- 設計思想に影響する判断は必ず記録
- 例外が3件以上増えたら思想レイヤーを見直す
