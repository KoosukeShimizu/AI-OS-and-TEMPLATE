# 1. 基本確認

- 何を検証するための正規表現か説明できるか
- OK例を2つ以上用意できるか
- NG例を2つ以上用意できるか

# 2. 過剰マッチ確認

- 想定外入力を許可していないか
- 空文字を誤って通していないか
- 改行や特殊文字で破綻しないか

---

# 3. 過少マッチ確認

- 正しい入力を弾いていないか
- 全角/半角の扱いは適切か

---

# 4. 危険パターン確認

- ネスト量が過剰でないか
- catastrophic backtrackingの恐れがないか
- 不要な .* を使っていないか
